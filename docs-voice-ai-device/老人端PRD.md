## 四、功能详情

### 4.1 语音对话

**描述**：老人与AI进行自然语音对话

**交互流程**：
1. 老人按住「按住说话」按钮录音
2. 松开按钮，语音转为文字发给AI
3. AI回复转为语音播放给老人

**技术实现**：
- STT：Whisper（openai-whisper）
- LLM：Ollama + gemma4:e4b
- TTS：espeak（本地，无需网络）

**对话示例**：
```
老人：你好
AI：您好，我是您的智能陪伴助手，有什么可以帮您的吗？

老人：提醒我每天8点吃降压药
AI：好的，我已经设置每天08:00提醒您吃降压药。

老人：我叫王大爷，今年72岁
AI：王大爷，我已记录您的信息，有任何需要随时告诉我。
```

### 4.2 跌倒检测 + 关怀流程

**描述**：检测老人跌倒，AI主动询问，无响应再联系子女

**完整流程**：

```
摄像头持续监测
    ↓
检测到跌倒姿态（躯干角度>45° + 高度下降>0.35）
    ↓
触发跌倒关怀流程：
    ↓
第1次询问："您没事吧？需要帮助吗？请说'没事'或'需要帮助'。"
    ↓ （等待20秒）
无响应 → 第2次询问："我有点担心您，需要我联系您的子女吗？请说'好的'联系，或'没事'继续。"
    ↓ （等待20秒）
无响应 → 第3次询问："最后一次询问，您真的没事吗？我要联系您的子女了，请说'好的'确认。"
    ↓ （等待20秒）
无响应 → 自动联系子女 + 120
```

**用户响应处理**：
- 说"没事"/"好的"/"没问题" → 确认无恙，继续监测
- 说"需要帮助"/"救命"/"打120" → 立即联系子女

**跌倒轨迹记录**：
- 跌倒事件记录最近30秒的姿态数据
- 包含：时间戳 + 姿态状态（站立/坐下/行走/躺下/跌倒）

### 4.3 用药提醒

**描述**：老人说"提醒我每天8点吃降压药"，AI自动创建提醒

**交互方式**：
- 语音创建：直接对AI说"提醒我每天X点吃XX药"
- 手动创建：通过UI的"用药"标签页添加

**提醒触发**：
- 到设定时间，TTS语音播报："王大爷您好，您该吃降压药了。"
- 记录用药日志（已提醒 / 已确认 / 跳过）

### 4.4 主动关怀

**描述**：AI定时主动问候，预防老人孤独感

**默认配置**：
- 每4小时问候一次
- 问候语示例："王大爷，您好！今天感觉怎么样？有什么需要帮忙的吗？"

**可配置项**：
- 开启/关闭
- 问候间隔（2/4/6/8小时）

## 六、技术架构

### 6.1 系统架构

```
┌──────────────────────────────────────────────────────────────┐
│                      UI 层 (Tkinter)                         │
│  录音按钮 | 跌倒告警 | 配置界面 | 对话历史 | 事件列表         │
└────────────────────────────┬─────────────────────────────────┘
                             │ HTTP
┌────────────────────────────▼────────────────────────────────┐
│                   FastAPI 后端 (端口8000)                      │
│  /chat  /stt  /tts  /health  /fall/*  /pose/*               │
│  /user/profile  /ai/profile  /medication/*  /care/*  /events  │
└────────────────────────────┬─────────────────────────────────┘
         ┌──────────────────┼──────────────────┐
         ▼                  ▼                  ▼
   ┌────────────┐    ┌────────────┐    ┌────────────┐
   │ Ollama     │    │ MediaPipe  │    │ espeak     │
   │ gemma4:e4b │    │ Pose       │    │ TTS        │
   └────────────┘    └────────────┘    └────────────┘
```

### 6.2 跌倒检测流程

```
摄像头(30fps) → VideoCapture
                          ↓
              MediaPipe Pose (每3帧检测1次)
                          ↓
              分析：躯干角度 + 头部高度变化
                          ↓ (置信度>0.6)
              触发跌倒关怀流程
                          ↓
              同时记录最近30秒姿态轨迹
                          ↓ (20秒无响应)
              HTTP POST 通知子女
```

### 6.3 数据存储

| 数据 | 存储方式 | 路径 |
|------|---------|------|
| 用户资料 | JSON文件 | `~/.voice-ai-device/user_profile.json` |
| AI称呼 | JSON文件 | `~/.voice-ai-device/ai_profile.json` |
| 用药提醒 | JSON文件 | `~/.voice-ai-device/medication_reminders.json` |
| 事件日志 | JSONL文件 | `~/.voice-ai-device/logs/events.jsonl` |

### 8.1 功能验收

| 测试项 | 通过标准 | 优先级 |
|--------|---------|--------|
| 语音对话 | 说出"你好"，AI响应并语音播放 | P0 |

### 8.2 非功能指标

| 指标 | 标准 |
|------|------|
| 对话响应时间 | LLM首token<3s（本地Ollama） |
| 跌倒检测延迟 | <2s（从跌倒动作到语音提醒） |
| TTS延迟 | <1s（espeak本地） |
| CPU占用 | <70%（1080p摄像头 + LLM推理） |
| 内存占用 | <4GB |
