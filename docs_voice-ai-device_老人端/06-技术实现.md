### 9.4 EmergencyContact模型

```python
class EmergencyContact(BaseModel):
    name: str           # 紧急联络人姓名
    relation: str       # 关系（配偶/儿子/女儿等）
    phone: str          # 电话
    wechat: str         # 微信
    is_primary: bool    # 优先联络人（互斥，只能一人）
```

**互斥逻辑**：
- 勾选某联络人为优先后，其他联络人的 `is_primary` 自动设为 False
- 联系亲属时默认使用优先联络人

### 9.5 事件历史多级展开实现

使用 Tkinter Treeview 组件实现按时间范围展开/折叠：

| 时间范围 | 显示方式 |

### 9.6 联络方式选择

### 9.7 用药提醒TTS消息格式

PRD【../05-数据与提醒.md-8.2触发流程】：TTS语音播报「王大爷您好，您该吃降压药了。」

实现使用 Piper TTS 中文神经网络声音（zh_CN-huayan-medium）：
```
{f"{user_name}您好，您该吃{medicine}了"}
```
无用户名时默认「大爷」。

> 注：Piper TTS 提供神经网络TTS，中文音色zh_CN-huayan-medium。
>
> **TTS准同步实现**：LLM流式输出token时，UI实时显示已生成的文字；TTS在后台线程运行，遇到句末标点（。！？）时触发整句合成，合成完成后播放语音。文字先于语音出现（<1s延迟），用户感知为同步。

### 9.8 语音唤醒模块

`voice_wakeup.py` 实现 PRD【../03-语音交互.md-4.2唤醒词检测】：
- 后台线程持续监听（openwakeword 唤醒词检测）
- 检测 AI 名字连续说两遍（如"小管家小管家"）
- 触发 `_wake_callback`，AI回复"我在"

### 9.9 虚拟硬件测试（voice-ai-device-raspberry-pi 分支）

支持无物理设备虚拟化测试，用于开发调试和 CI 自动化。

#### 9.9.1 虚拟音频（mock_audio.py）

`app/mock_audio.py` 提供 `AudioSource` 抽象基类和实现类：

| 类 | 说明 |
|---|---|
| `FileAudioSource` | 从 WAV 文件读取音频，模拟麦克风输入 |
| `SilenceAudioSource` | 产生静音，用于定时测试 |
| `ToneAudioSource` | 产生正弦波，用于音频路径测试 |

**使用方式**：
```python
from app.mock_audio import FileAudioSource

# 注入虚拟音频源
source = FileAudioSource(
    wav_files=["tests/fixtures/wake_xiaozhi_xiaozhi.wav"],
    loop=False
)

wakeup = VoiceWakeup(ai_name="小智")
wakeup.set_audio_source(source)  # 替换真实麦克风
wakeup.start()
```

#### 9.9.2 虚拟视频（mock_video.py）

`app/mock_video.py` 提供 `VideoSource` 抽象基类和实现类：

| 类 | 说明 |
|---|---|
| `FileVideoSource` | 从图像序列目录或视频文件读取帧，模拟摄像头 |
| `StaticImageSource` | 返回静态图像，用于单帧测试 |
| `SyntheticPoseSource` | 程序生成简单帧序列 |

**使用方式**：
```python
from app.mock_video import FileVideoSource

# 注入虚拟视频源
source = FileVideoSource(
    sources=["tests/fixtures/synthetic_poses/falling"],
    loop=False,
    frame_delay=0.001  # 快速播放
)

detector = FallDetector()
detector.set_video_source(source)
detector.run()  # 自动循环直到视频源耗尽
```

#### 9.9.4 合成素材局限性

MediaPipe Pose 依赖真实人体特征检测，合成 stick figure 图像无法可靠触发姿态检测。虚拟测试主要用于：
- 验证 pipeline 连接正确
- 测试超时/回调逻辑
- CI 自动化回归测试

真实跌倒检测验证需使用实际摄像头 footage。
