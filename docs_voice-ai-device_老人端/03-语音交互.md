## 四、语音交互

### 4.1 语音唤醒 + 对话流程

```
日常待机状态
    ↑
    │ 用户连续呼唤AI名字两遍（如"小管家小管家"）
    │
    ↓ 唤醒成功
AI立即回复"我在"
    ↓
进入对话状态
    ↓
用户说话 → faster-whisper → LLM → AI回复
    ↓
对话结束（如10秒无活动）→ 返回待机状态
```

**完整语音交互链路**：

```
唤醒词检测（openwakeword）
    ↓
语音活动检测（Silero VAD）← 检测是否有人说话
    ↓
语音转文字（faster-whisper）← 说话内容识别
    ↓
大语言模型思考回复（Qwen3-4B + llama.cpp）← AI思考和生成回复
    ↓
文字转语音（Piper TTS zh_CN-huayan-medium）+ 屏幕显示 ← LLM流式输出文字，UI实时显示；TTS后台线程合成，语音略晚于文字（准同步）
```

### 4.2 唤醒词检测

| 参数 | 值 | 说明 |
|------|-----|------|
| 唤醒词 | AI名字 × 2 | 如"小管家小管家" |
| 检测方式 | openwakeword + WebRTC VAD 前置过滤 | 实时监听，VAD过滤白噪声后送入openwakeword检测 |
| 响应延迟 | < 500ms | 唤醒后立即回复"我在" |
| 待机模式 | 屏幕熄灭，AI不主动响应 | 降低功耗；唤醒词、触屏、按钮操作均可点亮屏幕 |

**唤醒检测流程**：
1. WebRTC VAD（模式2）检测语音帧，过滤白噪声和安静环境
2. 缓冲约300ms音频后送入openwakeword
3. openwakeword计算唤醒词置信度分数
4. 双触发检测：两次检测间隔需>0.8秒（确保是两次独立发音）
5. 唤醒触发后3秒冷却，防止重复触发

**openwakeword配置**：
- 支持中文唤醒词模型（xiao_ai, xiao_bu, xiao_zhi）
- CPU占用低，适合边缘设备
- 模型存储：`models/openwakeword/`
- 唤醒阈值：0.0005（适配VAD过滤后的音频）

**对话模式**：
- 唤醒后进入持续对话状态
- Silero VAD检测语音，自动录音
- 说话后发送录音至STT转文字
- 10秒静音自动退出对话模式

### 4.3 AI自动填写

对话中AI自动识别意图并操作：

| 用户说 | AI自动 |
|--------|--------|
| "提醒我每天8点吃降压药" | 创建用药提醒 |
| "我叫张三" | 更新姓名 |
| "我68岁" | 更新年龄 |
| "提醒我下午2点吃消炎药" | 创建14:00用药提醒 |
