## 一、用户画像

- **主要用户**：独居老人（65岁以上），亲属不在身边
- **使用场景**：日常对话、跌倒检测紧急救援、用药提醒
- **操作能力**：可能不熟悉手机/电脑操作，语音交互为主
- **痛点**：孤独感、跌倒无人知、忘记吃药

---

## 二、解决方案

### 2.1 功能结构图

```
养老AI助手
├── 语音交互层
│   ├── 唤醒词检测（openwakeword）
│   ├── 语音活动检测（Silero VAD）
│   ├── 语音识别（faster-whisper）
│   ├── 大语言模型思考回复（Qwen3-4B + llama.cpp）
│   └── 语音合成（Piper TTS zh_CN-huayan-medium）
│
├── 通信层
│   ├── 微信电话（Webhook通知亲属APP）
│   ├── 普通电话（通过老人手机蓝牙/HTTP API触发）
│   └── 紧急通知
│       ├── 亲属（微信电话或普通电话）
│       └── 120（通过普通电话拨打）
│
├── 感知层
│   ├── 摄像头捕获（OpenCV）
│   ├── 姿态检测（MediaPipe Pose）
│   └── 跌倒检测（角度/高度分析）
│
├── 业务层
│   ├── 用户资料管理
│   ├── AI称呼初始化
│   ├── 紧急呼救流程
│   ├── 亲属通知（Webhook）
│   └── 用药提醒（定时任务）
│
└── 展示层
    ├── Tkinter UI（对话/事件/用药/设置）
    └── 跌倒告警弹窗
```

### 2.2 业务流程图

```
[老人动作]
     │
     ▼
┌─────────────────┐
│  语音唤醒/触发词  │ ←── 唤醒词（"小管家小管家"）
└─────────────────┘     或紧急触发词（"救命"/"打120"）
     │
     ▼
┌─────────────────┐
│   AI回复"我在"   │ ←── TTS语音合成（≤500ms响应）
└─────────────────┘
     │
     ├──→ [日常对话] → faster-whisper → LLM → AI回复
     │
     ├──→ [联系亲属] → Webhook通知亲属APP
     │
     ├──→ [紧急呼救] → 普通电话拨打120 + Webhook通知亲属APP
     │
     └──→ [用药提醒] → APScheduler定时触发 → TTS提醒
```

**跌倒检测业务流程：**

```
摄像头持续监控（30fps）
         │
         ▼
  MediaPipe Pose姿态分析
         │
    ┌────┴────┐
    │ 跌倒？  │ ← 躯干角度>45° + 头部高度下降>35%
    └────┬────┘
         │ Yes
         ▼
┌─────────────────┐
│ AI询问"您没事吧？"│
└─────────────────┘
         │
    ┌────┴────┐
    │ 有响应？ │
    └────┬────┘
     Yes │   │ No
         ▼   └──→ [3次询问无响应] → 自动拨打120
   继续监测

联系亲属流程：
用户说"联系亲属" → AI确认 → 微信电话（Webhook通知亲属APP）或普通电话（蓝牙）

**通信矩阵**：

| 触发事由 | 对象 | 途径 | 说明 |
|---------|------|------|------|
| 日常联络 | 亲属 | 微信电话或普通电话 | 用户主动说"联系亲属" |
| 紧急呼救 | 亲属 | 微信电话或普通电话 | AI询问后用户选择 |
| 紧急呼救 | 120 | 普通电话 | AI询问后用户选择，或3次无响应自动拨打 |

用药提醒流程：
用户说"提醒我每天8点吃降压药" → LLM解析 → APScheduler创建任务
                                                    │
                                                    ▼
                                           定时触发 → TTS语音提醒
```

### 2.4 本地离线语音交互技术路径

**技术选型：完全本地化、离线运行**

语音交互流程：
```
唤醒词检测（openwakeword）
    ↓
语音活动检测（Silero VAD）← 检测是否有人说话
    ↓
语音转文字（faster-whisper）← 说话内容识别
    ↓
大语言模型思考回复（Qwen3-4B + llama.cpp）← AI思考和生成回复
    ↓
文字转语音（Piper TTS zh_CN-huayan-medium）+ 屏幕显示 ← LLM流式输出文字，UI实时显示；TTS后台线程合成，语音略晚于文字（准同步）
```

| 模块 | 技术方案 | 说明 |
|------|---------|------|
| 唤醒词检测 | **openwakeword** | 预训练唤醒词模型，开箱即用，CPU占用低 |
| 语音活动检测 | **Silero VAD** | 语音活动检测，判断何时开始/结束说话 |
| 语音识别(STT) | **faster-whisper (whisper.cpp)** | CPU优化INT8量化，支持中文 |
| 大语言模型(LLM) | **Qwen3-4B + llama.cpp** | 4B参数量化版本，本地离线运行 |
| 语音合成(TTS) | **Piper TTS** (zh_CN-huayan-medium) | 神经网络TTS，中文音色 |

**离线架构优势：**

- 无需网络连接，保护老人隐私
- 响应速度快，无网络延迟
- 适合树莓派等边缘设备部署

**组件依赖：**

```
voice-ai-device/
├── app/
│   ├── voice_wakeup.py      # openwakeword 唤醒词检测
│   ├── vad.py               # Silero VAD 语音活动检测
│   ├── stt.py               # faster-whisper 语音识别
│   ├── llm_client.py        # 统一LLM客户端
│   ├── llm.py               # Qwen3-4B + llama.cpp
│   ├── tts.py               # Piper TTS
│   └── ...
├── requirements.txt
```

**性能指标目标：**

| 指标 | 目标值 |
|------|--------|
| 唤醒响应 | < 500ms |
| VAD检测延迟 | < 100ms |
| 对话首token | < 3s（Qwen3-4B） |
| TTS延迟 | < 1s |
| 内存占用 | < 4GB |
