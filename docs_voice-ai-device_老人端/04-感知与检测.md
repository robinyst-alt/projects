## 五、跌倒检测算法

### 5.1 检测逻辑

使用 MediaPipe Pose 分析躯干角度和头部高度：

```python
# 关键阈值
FALL_ANGLE_THRESHOLD = 45.0  # 躯干与垂直方向夹角 > 45° 视为跌倒
HEIGHT_DROP_RATIO = 0.35     # 高度下降 > 35% 视为跌倒
CONFIDENCE_THRESHOLD = 0.6    # 姿态检测置信度 > 0.6 才采纳
```

### 5.2 跌倒判定条件

同时满足以下条件判定为跌倒：
1. 躯干角度 > 45°（身体倾斜）
2. 头部高度下降 > 35%（相对于站立基准）
3. 持续时间 > 0.5秒（排除短暂弯腰）

### 5.3 姿态分类

| 条件 | 姿态 |
|------|------|
| 躯干角度 > 60° | 【躺着】 |
| 躯干角度 > 45° 且膝盖与臀部高度接近 | 【坐着】 |
| 躯干角度 > 45° 且不符合坐着条件 | 【跌倒】 |
| 脚踝高度周期性变化 > 0.08 | 【行走】 |
| 脚踝低于臀部 > 0.15 | 【站立】 |

## 六、呼救流程

### 6.1 框架概述

呼救流程按**触发类型**分为两类：

| 触发类型 | 说明 | 详见 |
|---------|------|------|
| **被动触发** | MediaPipe检测到跌倒姿态 | 【../04-感知与检测.md-5.跌倒检测算法】 |
| **主动触发** | 用户喊出紧急呼救触发词 | 【6.3主动触发：紧急呼救流程】 |

按**Action类型**分为两种：

| Action类型 | 说明 | 详见 |
|-----------|------|------|
| **联系亲属** | Webhook通知亲属 + 电话拨打 | 【6.4联系亲属流程】 |
| **拨打120** | 直接拨打120急救电话 | 【6.5拨打120流程】 |

AI询问用户时给出选择：「是要我联系您的亲属，还是直接拨打120？」

**接通时AI简报**：无论联系亲属还是拨打120，接通后AI自动播报简要情况。

**AI简报参数说明**（全局）：

| 参数 | 来源 | 说明 |
|------|------|------|
| {地址} | 【../02-界面设计.md-3.6设置页老人信息-地址】 | 老人住址 |
| {姓名} | 【../02-界面设计.md-3.6设置页老人信息-姓名】 | 老人姓名 |
| {年龄} | 【../02-界面设计.md-3.6设置页老人信息-年龄】 | 老人年龄 |
| {病史} | 【../02-界面设计.md-3.6设置页老人信息-病史】 | 老人病史（无病史则省略） |
| {,已无意识} | - | 用户20秒内无响应时追加 |
| {关系} | 【../02-界面设计.md-3.6设置页紧急联络人-关系】 | 亲属关系（配偶/儿子/女儿等） |
| {亲属姓名} | 【../02-界面设计.md-3.6设置页紧急联络人-姓名】 | 紧急联络人姓名 |
| {亲属电话} | 【../02-界面设计.md-3.6设置页紧急联络人-电话】 | 紧急联络人电话 |
| {用户触发词描述} | 用户喊出的触发词 | 触发词对应的紧急情况描述 |

**通信矩阵**：

| 触发事由 | 对象 | 途径 | 说明 |
|---------|------|------|------|
| 日常联络 | 亲属 | 微信电话或普通电话 | 用户主动说"联系亲属" |
| 紧急呼救 | 亲属 | 微信电话或普通电话 | AI询问后用户选择 |
| 紧急呼救 | 120 | 普通电话 | AI询问后用户选择，或3次无响应自动拨打 |

**触发词映射**：

| 触发词 | {用户触发词描述} |
|--------|------------------|
| 救命 | 紧急求助 |
| 打120 | 需要急救 |
| 紧急呼救 | 紧急呼救 |
| 我摔倒了 | 刚发生跌倒 |
| 快来人 | 紧急求助 |

### 6.2 被动触发：跌倒检测流程

跌倒判定基于MediaPipe Pose（详见【5.跌倒检测算法】）。

```
检测到跌倒（MediaPipe Pose 确认）
    ↓
记录事件 + 轨迹（最近30秒姿态数据，详见【../05-数据与提醒.md-7.运动轨迹记录】）
    ↓
立即触发语音唤醒
"我在"（≤2秒）
    ↓
"您没事吧？"（≤2秒）
    ↓ （等待20秒）
无响应 → "您摔倒了吗？是要我联系您的亲属，还是直接拨打120？"
    ↓ （等待20秒）
无响应 → "我准备拨打120了。"
    ↓ （等待20秒）
无响应 → 拨打120 + 记录事件【紧急呼救】
```

**用户响应处理**：

| 响应 | 含义 | 处理 |
|------|------|------|
| 联系亲属 | 选择Action | 联系亲属（执行【6.4联系亲属流程】） |
| 打120 | 选择Action | 拨打120（执行【6.5拨打120流程】） |
| 静默/无响应 | 超时 | 3次询问后自动拨打120 |

### 6.3 主动触发：紧急呼救流程

紧急呼救**本身即唤醒词**，无需先唤醒AI。

**触发词**："救命"、"打120"、"紧急呼救"、"我摔倒了"、"快来人"等。

```
用户喊出触发词
    ↓
AI立即回复（≤1秒）
"我在，您哪里不舒服？"
    ↓ （等待20秒）
有响应 → 按用户选择执行对应Action
无响应 → "您哪里不舒服？是要我联系您的亲属，还是直接拨打120？"
    ↓ （等待20秒）
无响应 → "我准备拨打120了。"
    ↓ （等待20秒）
无响应 → 拨打120 + 记录事件【紧急呼救】
```

**用户响应处理**：

| 响应 | 含义 | 处理 |
|------|------|------|
| 是的/确认/好 | 确认紧急 | 执行用户选择的Action |
| 没事/不用了/cancel | 撤销 | 取消流程 |
| 联系亲属 | 选择Action | 联系亲属（执行【6.4联系亲属流程】） |
| 打120 | 选择Action | 拨打120（执行【6.5拨打120流程】） |

### 6.4 联系亲属流程

```
AI复述
"好的，我使用{微信电话/电话}打给{优先联络人姓名}，请稍后。"
    ↓
执行联络
├── 若选择"微信电话" → Webhook通知亲属APP（微信电话）
└── 若选择"普通电话" → 通过老人手机蓝牙触发拨打电话
    ↓
AI简报（接通时自动播报）：
├── 微信电话：「您好，{姓名}{用户触发词描述}，需要帮助。」
└── 普通电话：「您好，{姓名}{用户触发词描述}，需要帮助。」
    ↓
记录事件【联系亲属：xxx】（详见【../02-界面设计.md-3.5事件历史】）
```

**AI简报触发条件**：无论选择微信电话还是普通电话，接通后均自动播报AI简报。

### 6.5 拨打120流程

```
AI复述
"好的，我现在拨打120，请稍后。"
    ↓
执行联络
├── Webhook通知亲属APP（紧急通知）
└── 通过老人手机蓝牙拨打120
    ↓
AI简报（接通时自动播报）：
「医生你好，住在{地址}的{姓名}{用户触发词描述}{,已无意识}，需要帮助。其{关系}{亲属姓名}叫{亲属姓名}，联系电话是{亲属电话}。他今年{年龄}岁，具有{病史}。谢谢。」
    ↓
记录事件【紧急呼救】（详见【../02-界面设计.md-3.5事件历史】）
```

**AI简报触发条件**：拨打120接通后自动播报完整AI简报，包含老人位置、健康信息、紧急联络人信息。

### 6.6 关键时序要求

| 阶段 | 时限 |
|------|------|
| 跌倒确认 → "我在" | ≤ 2秒 |
| "我在" → 第1次询问 | ≤ 2秒 |
| 每次询问等待 | 20秒 |
| 3次询问无响应 → 触发Action | 60秒（3×20秒） |

### 6.7 流程状态

| 状态 | 说明 |
|------|------|
| IDLE | 无活动 |
| CHECKING_IN | 正在询问（1-3次） |
| ACTIONING | 正在执行联络 |
| RESOLVED | 已解决 |
