## 三、语音交互

### 3.1 语音输入流程

```
老人按住「按住说话」按钮
    ↓
录音（sox rec，16kHz，mono，16-bit）
    ↓
松开按钮，发送音频到 /stt
    ↓
Whisper 转文字
    ↓
发送文字到 /chat
    ↓
Ollama gemma4:e4b 生成回复
    ↓
espeak TTS 合成语音
    ↓
播放音频（afplay/aplay）
```

### 3.2 AI自动填写

对话中AI自动识别意图并操作：

| 用户说 | AI自动 |
|--------|--------|
| "提醒我每天8点吃降压药" | 创建用药提醒 |
| "我叫张三" | 更新姓名 |
| "我68岁" | 更新年龄 |
| "提醒我下午2点吃消炎药" | 创建14:00用药提醒 |

## 四、跌倒检测算法

### 4.1 检测逻辑

使用 MediaPipe Pose 分析躯干角度和头部高度：

```python
# 关键阈值
FALL_ANGLE_THRESHOLD = 45.0  # 躯干与垂直方向夹角 > 45° 视为跌倒
HEIGHT_DROP_RATIO = 0.35     # 高度下降 > 35% 视为跌倒
CONFIDENCE_THRESHOLD = 0.6    # 姿态检测置信度 > 0.6 才采纳
```

### 4.2 跌倒判定条件

同时满足以下条件判定为跌倒：
1. 躯干角度 > 45°（身体倾斜）
2. 头部高度下降 > 35%（相对于站立基准）
3. 持续时间 > 0.5秒（排除短暂弯腰）

### 4.3 姿态分类

| 条件 | 姿态 |
|------|------|
| 躯干角度 > 60° | 【躺着】 |
| 躯干角度 > 30° 且膝盖与臀部高度接近 | 【坐着】 |
| 躯干角度 > 30° 且不符合坐着条件 | 【跌倒】 |
| 脚踝高度周期性变化 > 0.08 | 【行走】 |
| 脚踝低于臀部 > 0.15 | 【站立】 |

## 五、跌倒关怀流程

### 5.1 完整流程

```
检测到跌倒
    ↓
记录事件 + 轨迹（最近30秒姿态数据）
    ↓
第1次询问（立即）
"您没事吧？需要帮助吗？请说'没事'或'需要帮助'。"
    ↓ （等待20秒）
无响应 → 第2次询问
"我有点担心您，需要我联系您的子女吗？请说'好的'联系，或'没事'继续。"
    ↓ （等待20秒）
无响应 → 第3次询问
"最后一次询问，您真的没事吗？我要联系您的子女了，请说'好的'确认。"
    ↓ （等待20秒）
无响应 → 触发SOS：Webhook通知子女 + TTS播报"已联系您的子女"
```

### 5.2 用户响应关键词

| 关键词 | 含义 | 处理 |
|--------|------|------|
| 没事、我没事、好的、没问题、OK | 用户确认无恙 | 记录并继续监测 |
| 需要帮助、救命、帮帮我、打120 | 用户需要帮助 | 立即联系子女 |
| cancel | 用户取消 | 取消当前关怀流程 |

### 5.3 关怀流程状态

| 状态 | 说明 |
|------|------|
| IDLE | 无活动 |
| CHECKING_IN | 正在询问老人（1-3次） |
| ESCALATING | 正在联系子女 |
| RESOLVED | 已解决（用户确认OK或已取消） |

## 六、运动轨迹记录

### 6.1 数据结构

```python
# 滚动缓冲区：最近5分钟（300条）
MAX_TRAJECTORY_BUFFER = 300

{
    "timestamp": 1745337580.123,  # Unix时间戳
    "pose": "standing"            # 姿态状态
}
```

### 6.2 轨迹用途

- 跌倒事件回溯：跌倒发生前后各30秒的姿态序列
- 事件日志存储：跌倒事件包含 `trajectory` 字段
- 查询API：`GET /pose/trajectory?seconds=30`
