# 养老AI助手 - 子女端技术PRD

> 版本：v1.0
> 整理时间：2026-04-25
> 状态：规划阶段

---

## 一、用户画像

- **主要用户**：子女（老人亲属），35-55岁
- **使用场景**：接收老人设备推送的消息、查看老人状态、远程配置设备
- **设备**：智能手机（iOS/Android）
- **痛点**：担心老人独自在家安全、无法及时了解老人状况

---

## 二、产品定位

| 维度 | 说明 |
|------|------|
| 产品名称 | 养老AI助手（子女端） |
| 核心定位 | 老人设备的消息接收方和远程管理端 |
| 与老人设备关系 | 老人设备的消息接收方和远程管理端 |

---

## 三、核心功能

### 3.1 功能优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 消息接收 | 接收老人设备的Webhook推送 |
| P0 | 事件列表 | 显示老人所有事件历史 |
| P0 | 事件详情 | 查看跌倒事件包含的轨迹数据 |
| P1 | 设备配置 | 远程管理老人设备参数 |
| P1 | 用药提醒管理 | 查看/新增/删除老人的用药提醒 |
| P2 | 双向消息 | 给老人发送文字/语音 |
| P2 | 每日健康报告 | 汇总一天的活动轨迹 |

### 3.2 推送消息类型

| 消息类型 | 触发时机 | 内容 |
|---------|---------|------|
| fall_detected | 检测到跌倒且AI询问无响应 | 老人姓名、位置、跌倒置信度、轨迹 |
| sos_triggered | 老人触发SOS | 老人姓名、位置 |
| medication_reminder | 用药提醒触发 | 药品名称、提醒时间 |
| care_reminder | 主动关怀触发 | 问候消息 |

## 四、UI/UX设计

### 4.1 页面结构

```
子女APP
├── 首页/消息列表（默认）
│   ├── 紧急告警横幅（跌倒/SOS时置顶显示）
│   └── 事件卡片列表（按时间倒序）
│
├── 事件详情
│   ├── 事件基本信息（类型、时间、老人信息）
│   ├── 跌倒轨迹（如果有，包含姿态时间线）
│   └── 操作按钮（确认已处理/回拨/联系120）
│
├── 设备配置
│   ├── 老人信息（姓名、房间位置）
│   ├── 紧急联系人（最多3个）
│   ├── 跌倒检测灵敏度
│   ├── 用药提醒列表（增删改）
│   └── AI称呼
│
└── 我的
    ├── 账号信息
    ├── 消息通知设置
    └── 关于/帮助
```

### 4.2 紧急告警横幅

当收到跌倒/SOS消息时，首页顶部显示醒目横幅：

- 背景色：红色（跌倒）/ 橙色（SOS）
- 显示：老人姓名 + 事件类型 + 时间
- 操作：点击查看详情 / 一键拨打

### 4.3 事件卡片

```
┌─────────────────────────────────┐
│ 🤕 跌倒检测                      │
│ 王大爷 · 客厅 · 10:30           │
│ 置信度 85% · 已自动联系子女     │
└─────────────────────────────────┘
```

点击展开显示轨迹时间线：

```
10:27 【站立】
10:28 【行走】
10:29 【行走】
10:30 【跌倒】← 跌倒时刻
10:30 【躺着】
```

## 五、技术实现

### 5.1 技术选型

| 组件 | 选型 | 说明 |
|------|------|------|
| 跨平台框架 | 微信小程序 / React Native | 待讨论 |
| 消息接收 | Webhook回调 + 轮询混合 | 优先Webhook，失败时轮询 |
| 状态存储 | 本地缓存 + 云端同步 | 支持离线查看历史 |

### 5.2 数据流

```
老人设备                          子女APP
    │                                │
    │  ──POST Webhook───────────>   │  接收推送
    │                                │
    │  <──GET /events───────────    │  拉取事件
    │                                │
    │  <──PUT /user/profile─────    │  远程配置
    │                                │
    │  <──POST /device/message──    │  发送消息
```

### 5.3 Webhook回调格式

```json
POST {注册的Webhook URL}
{
  "event_type": "fall_detected",
  "device_id": "device-001",
  "elder_name": "王大爷",
  "location": "客厅",
  "timestamp": "2026-04-25T10:30:00Z",
  "fall_confidence": 0.85,
  "trajectory": [
    {"timestamp": 1745337580.1, "pose": "standing"},
    {"timestamp": 1745337581.2, "pose": "falling"},
    {"timestamp": 1745337582.3, "pose": "lying_down"}
  ],
  "ai_response": "检测到跌倒，已联系您的子女"
}
```

## 六、API对接（子女端视角）

### 6.1 调用的老人设备API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/events` | 获取事件列表 |
| GET | `/events/{id}` | 获取事件详情（含轨迹） |
| GET | `/user/profile` | 获取老人信息 |
| PUT | `/user/profile` | 更新老人信息 |
| GET | `/medication/reminders` | 获取用药提醒 |
| POST | `/medication/reminders` | 添加用药提醒 |
| DELETE | `/medication/reminders/{id}` | 删除用药提醒 |
| GET | `/care/settings` | 获取关怀设置 |
| PUT | `/care/settings` | 更新关怀设置 |
| POST | `/device/message` | 给老人发送文字/语音 |

### 6.2 子女端注册Webhook

```json
POST /notification/settings
{
  "webhook_url": "https://child-app.example.com/webhook/alert",
  "enabled": true,
  "retry_count": 3
}
```
